# チャット画面レイアウト仕様

- 版: v0.2（2026-07-13, Fable 改訂 — FR-7 対応でセッションサイドバーを追加（§4））
- v0.1（2026-07-11, Fable 起草 — 利用者フィードバック「AI の回答を画面中央に配置せよ」対応）
- 参照: [OC_guidanceLLM](https://github.com/takahashiJe/OC_guidanceLLM) の `ChatMessage.vue`

## 1. メッセージ配置の原則

| 要素 | 配置 |
|---|---|
| ユーザー発言 | **右寄せ**の吹き出し（現状維持: `flex justify-end` + 白吹き出し） |
| AI 回答 | **中央カラム**。吹き出しにせず、`max-w-3xl mx-auto` のドキュメント風ブロックとして全幅中央に置く（参照実装の `max-w-4xl mx-auto` と同思想。本システムは全体を max-w-3xl で統一） |
| pending 演出（Aurora Ring） | AI 回答と同じ中央カラムの左端から始まる行 |
| 出典 | AI 回答ブロック内の下部（中央カラム内） |

- 「LINE 型の左右対話」ではなく「AI の回答が主役のドキュメント型」にする。
- モバイル幅では実質全幅（px-4）だが、タブレット/PC 幅で AI 回答が**画面中央**に来ることが要件。
- ユーザー発言の右寄せは中央カラムの右端に揃える（コンテナは共通の `max-w-3xl mx-auto`）。

## 2. 実装ノート

- 現状の `ChatView.vue` は AI メッセージを `justify-start`（左寄せ）にしている。これを廃止し、
  AI メッセージは行コンテナ自体を全幅にして内部ブロックを `mx-auto` にする。
- LoadingSpinnerV5 の morph・スクロールクリアランス等の既存挙動は維持すること。
- ヘッダー・入力欄は現状のダークテーマを維持（今回の変更対象はメッセージ配置のみ）。

## 3. 受け入れチェックリスト

- [ ] PC 幅（≥1024px）で AI 回答が画面中央のカラムに表示される
- [ ] ユーザー発言は同カラムの右端に寄る
- [ ] pending 演出・morph・自動スクロールが退行していない
- [ ] モバイル 390px で従来どおり読みやすい

## 4. セッションサイドバー（FR-7, 2026-07-13 追加）

### 4.1 レイアウト

- **PC（lg: ≥1024px）**: 画面左に常設サイドバー（幅 260〜288px、ダークテーマ継続）。右側にチャット本体。
  サイドバーは折りたたみボタンで開閉できるとなお良い（必須ではない）。
- **モバイル（<1024px）**: サイドバーは非表示。ヘッダー左のハンバーガーで左からドロワー
  （オーバーレイ + スライドイン、背景タップで閉じる）。スレッド選択・新規会話で自動的に閉じる。

### 4.2 サイドバーの内容

1. 「新しい会話」ボタン（最上部・常に表示）
2. スレッド一覧（`GET /api/threads`、updated_at 降順）。各項目: タイトル（1 行 truncate）。
   現在開いているスレッドをハイライト。
3. 各項目のメニュー（ホバー/タップで表示）: 「名前を変更」（インライン編集 or プロンプト → `PATCH`）、
   「削除」（確認後 `DELETE`。現在表示中なら新規会話状態へ遷移）
4. 最下部にユーザー名とログアウト（ヘッダーから移してもよい）

### 4.3 ルーティングと復元

- ルートを `/chat/:threadId?` にする。
  - `threadId` あり: マウント時に `GET /api/threads/{id}` で履歴を復元（sources 含む）。404 なら `/chat` へ。
  - なし: 空の新規会話画面。
- 新規会話は最初の `done` イベントで `thread_id` が確定した時点で `router.replace('/chat/{id}')` し、
  サイドバー一覧の先頭に追加（または一覧を再取得）する。
- リロードしても同じスレッドが同じ内容で表示されること（FR-7 の受け入れ条件）。
- 送信中（SSE ストリーム中）のスレッド切替は、進行中メッセージを中断しない範囲で許可
  （最小実装: 送信中はサイドバーの項目を disabled にしてよい）。

### 4.4 受け入れチェックリスト

- [ ] リロードで会話が消えない（同 URL で復元される）
- [ ] PC で常設サイドバー、モバイルでドロワーが機能する
- [ ] 新規会話 → 回答完了で URL がスレッド URL に置き換わり、一覧の先頭に現れる
- [ ] スレッド切替・名前変更・削除が動作し、他ユーザーのスレッドには 404
- [ ] 既存の pending 演出・自動スクロール・中央カラム配置（§1〜2）が退行していない
