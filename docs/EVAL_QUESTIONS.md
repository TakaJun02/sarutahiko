# 検収用サンプル質問集（RAG 評価）

- 版: v0.1（2026-07-11, Fable 作成）
- 用途: Phase 4 ナレッジ検収・Phase 6 統合テストで使用。
  各質問に対し「適切なナレッジファイルが検索で当たるか」「回答に出典が付くか」「誤った断定をしないか」を確認する。

## G. 回帰質問（v5・2026-07-12 実運用の失敗事例。ハーネス v5 §V5-6 の検収に使用）

- 26. 「サイバーフィジカルシステム研究室では、どんな出展がありますか？」
      → `lab-cps-open-lab-2026` の出展一覧（人間タワーバトル等）と時間帯・場所（大学院棟D404）を回答できること。
- 27. 「サイバーフィジカルシステム研究室の学生メンバーについて教えてください」
      → `lab-cps-members` の修士・学部生の氏名を含めて回答できること。
- 28. 「サイバーフィジカルシステム研究室の先生は誰ですか？」（v3 §9 の継続。山口高康教授）

## H. 経路案内（FR-11・2026-07-13 追加。docs/ROUTE_GUIDANCE.md §6 の検収に使用）

- 29. 「情報工学科の出展はどこでやっていますか？」
      → 学部棟Ⅰ G1ゾーン・大学院棟 Dゾーン・共通棟 K321 を棟・ゾーン付きで回答できること。
- 30. 「GI512 はどこにありますか？」
      → 学部棟Ⅰ G1ゾーン 5階（情報ネットワーク研究室）と回答できること（小文字・全角表記でも同様）。
- 31. 「総合受付はどこですか？」
      → 収集した出典に基づき場所を特定。出典が無ければ正直に述べて相談先を案内できること。
- 32. 「サイバーフィジカルシステム研究室の公開はどの部屋ですか？」
      → 大学院棟 D404 と回答できること。
- 33. 「羽後本荘駅からキャンパスまでの行き方と、着いてからの流れを教えてください」
      → 既存 access ナレッジ（バス路線・徒歩）＋受付情報を組み合わせて回答できること。
- 34. 「食堂はどこにありますか？」
      → 棟・ゾーンまで特定して回答できること。
- 35. 「食堂から D404 までの経路を教えてください」（v0.2・2026-07-14 追加）
      → カフェテリア（メディア交流棟）→ 学部棟Ⅱ（またはⅠ）2階（2階が地上）→ 連絡通路で大学院棟2階 →
      エレベーター/階段、の経由地列挙で回答できること。
- 36. 「総合受付から学部棟Ⅰへはどう行きますか？」（v0.2 追加）
      → 共通施設棟（K）から学部棟Ⅰ2階へのつながり（ROUTE_GUIDANCE.md §7.2 E5）で案内できること。
- 37. 「体育館へはどう行けばいいですか？」（v0.2 追加）
      → カフェテリア方面の通路経由＋屋内運動施設1階、で案内できること。
- 共通の不合格条件: 根拠にない道順（曲がる方向・徒歩分数）の創作、裁定済み経路グラフにない連絡通路・階の捏造、
  場所を特定せず「当日スタッフにお尋ねください」だけで返す丸投げ。

## I. 経路マップカードとマップタップ（FR-26・2026-07-17 追加。docs/MAP_CARD.md §9 と同期）

- 38. 「食堂から D414 への行き方は？」
      → `mode: route`。path は `cafeteria → E6a → g1 → E1 → d`（または並行エッジ規則に従う
      g2 系）で、4階の連絡通路を選ぶ。ステップに「学部棟Ⅰ（Ⅱ）は2階が地上」
      「エレベーターまたは階段で4階へ」「4階の連絡通路」を含む。
- 39. 「食堂から D404 への経路は？」
      → `mode: route`。D404 の階バッジ・到着階ステップを出さず、
      「階は館内表示・当日スタッフでご確認ください」で終わる。
- 40. 「D414 に行きたい」（新規スレッド）
      → `mode: ask_origin` と定型短文でターン終了。カフェテリアをタップすると
      「現在地はカフェテリア（食堂）です。D414 に行きたい」を通常送信し、38 と同等の route カードを返す。
- 41. 40 の後、同じスレッドで「じゃあ体育館は？」
      → 履歴からカフェテリア発の E7 route カードを返し、回答冒頭に出発地を明示する。
- 42. 「GI512 はどこ？」
      → `mode: place`。g1 をハイライトし「GI512・5階」バッジを表示する。本文回答は従来どおり。
- 43. 「総合受付から学部棟Ⅱへ」
      → E5+E6c の前面通路系を選び、連絡通路2本の屋内迂回にしない。
- 44. 「羽後本荘駅からの行き方」
      → map イベントなし。従来テキストのみでイベント列にも回帰がない。
- 45. UI 検収
      → SVG と等価なノード名チップ列、44px 以上のタップ領域、最新 ask_origin のみ活性、
      reduced-motion 遵守、375px 幅で横スクロールなし、履歴リロードで route/place を再現し
      ask_origin は非活性になる。
- 46. 回帰
      → backend の pytest 全件、frontend の既存テストランナー全件、`npm run build` がすべて成功する。
