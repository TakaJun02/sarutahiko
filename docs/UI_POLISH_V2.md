## 0. 発端（利用者指示 2026-07-14）

> UIをブラッシュアップしてください。単に色を多用して派手にするのではなく、かっこいい見た目にしてください。
> いかにもAIが作ったUIという雰囲気がします。センスの感じるカッコ良いアプリUIを作成してください。
> 個人的にはGeminiアプリのUIがスタイリッシュでセンスの感じるかっこいいUIだと感じます。
> 静止画の見た目だけでなく、アニメーションや演出もこだわってください。
> ログイン画面と会話画面の両方ブラッシュアップしてください。
> ※このシステムはPCでもスマホでも使用します。両方で最適化された見た目にしてください。
> デザインについては、CodexでGPT-5.6Solがメインで動いてください。

## 1. 現状の課題（Fable の分析、2026-07-14 実機スクリーンショットより）

1. **ブランド不在**: ログイン画面にロゴ・アプリ名・大学名が一切ない。何のアプリか画面から分からない。
2. **PC ログインの間延び**: 1440px で白面が画面の 2/3 を占めて空虚。黒シートは全幅に伸び、max-w-md の
   フォームだけが黒い海に浮く。モバイル（390px）では良く見える構図が、PC で設計されていない。
3. **アクセントの散乱**: 3 色グラデ（#FF8A65→#FFEB3B→#69F0AE）がヘッダー下ヘアライン・カウントダウン
   チップ枠・アクティブスレッドバー・アバター・送信ボタン・入力欄のミント発光…と多所に出現。
   「一本の糸」の意図（FR-9）が実装では「虹の散乱」になっており、利用者の言う「色を多用して派手」の主因。
4. **ユーザー吹き出しがほぼ純白**: ダーク画面で眩しく浮く。Gemini のダークテーマはユーザー発言も
   ソフトな面色（明度を落としたグレー）＋白文字で、画面全体の明度が静か。
5. **タイポグラフィの個性ゼロ**: 全要素が Noto Sans の似たウェイト・サイズ階層。数字（カウントダウン・
   時刻表・「2026」）にも表情がない。
6. **演出の不足**: ローディングリング（確定仕様）以外に演出がほぼない。画面遷移はぶつ切り、空状態・
   メッセージ出現・ログイン成功時に何も起きない。
7. 総評: 破綻はないが「量産型ダーク AI チャット」テンプレの域を出ない。これが「いかにも AI が作った UI」
   の正体と分析する。

## 2. 不可侵の制約（MUST — これだけは破らない）

1. **LoadingSpinnerV5（Aurora Ring）**: 動作・タイミング・ステップ別テーマ色・morph（pending→settled）は
   `docs/UI_LOADING_ANIMATION.md` の確定仕様。リング周囲の「見せ方」（余白・整列・背景との調和）は調整可、
   リング自体（SVG 構造・アニメーション値・STEP_THEMES・実況テキストの shimmer）は不可侵。
   Q-013 の `type="transition"` 修正も維持する。
2. **ログインの骨格**: 「ライトエリア（タイプライター見出し＋点滅カーソル）＋下部シート」の 2 層構造、
   タイプライターの 8 フレーズと打鍵タイミング（150/75/2000/500ms）、パスワードレス（ニックネームのみ）は
   変更不可。それ以外 — シートの色・角丸・内部レイアウト、ライトエリアへの要素追加（ブランド行・背景表情）、
   両エリアの面積比・PC での構図 — は Sol の裁量（`docs/UI_LOGIN.md` v0.4 で緩和済み）。
3. **チャットの中央カラム原則**（`docs/UI_CHAT.md` §1: AI 回答=中央ドキュメント型 / ユーザー発言=右寄せ
   吹き出し）とサイドバーの機能要件（§4）。※ユーザー吹き出しの「白」は v0.3 で解除済み — 色は Sol の裁量。
4. **表現層のみ変更**: SSE スキーマ・stores（auth/chat）の公開 API・router のルート定義・backend は
   変更しない。`chat.spec.js` が壊れる store 変更は不可。
5. **アクセシビリティ**: タップターゲット 44×44px 以上、コントラスト本文 4.5:1・補助 3:1 以上、
   全インタラクティブ要素に focus-visible リング、`prefers-reduced-motion` で装飾モーション全停止（機能は保つ）。
6. **オフライン堅牢性**: 実行時に外部ネットワークへ出る資産（CDN フォント・外部画像・外部 CSS/JS）を
   追加しない。全アセットはリポジトリ同梱でビルドに含める（会場ネットワークが不安定でも崩れないこと）。
7. **新規 npm 依存の追加は不可**（アニメーションライブラリ含む）。演出は CSS / SVG / Vue 標準の
   `<Transition>` / `<TransitionGroup>` で実現する。
8. `npm test` と `npm run build` が通ること。FR-1〜11 の機能退行なし。

## 4. デザイン方向 — Fable の推奨（Sol は上書き可）

**コンセプト案「Quiet Aurora」**: 画面の 98% を無彩色のダークニュートラルに沈め、ブランドのオーロラは
「AI が生きている瞬間」にだけ現す。

- オーロラの出現箇所を最大 3 つに制限する:
  1. ローディングリング（確定仕様そのまま）
  2. チャット空状態の挨拶テキストのグラデーション（Gemini の "Hello, ○○" に相当する一撃）
  3. 入力欄の streaming 中にだけ現れる微かな縁の光
- 排除する: ヘッダー虹ヘアライン / アクティブスレッド虹バー / 入力欄ミント発光 / 虹枠チップ / 3 色アバター。
  置き換えは無彩色（面・ボーダー・白）＋必要なら単色アクセント 1 色まで。
- テキスト用グラデは純黄 #FFEB3B を経由しない補正系が上品（例: #FF8F6D → #FFC46B → #6FE8A8）。
  リングの STEP_THEMES とは別トークンとして定義する（リング側は変更不可）。
- ユーザー吹き出しは白をやめ、Gemini ダークテーマ風のソフトな面色（例: raised より一段明るい無彩面）＋白文字。
- ニュートラル階調はごく僅かに暖色へ寄せると、アプリアイコン（オレンジの歯車・赤い Y・緑の CPS lab）と喧嘩しない。
- 数字・欧文にディスプレイフォント（§3）。カウントダウンの「5」、時刻表の「10:00」、「AI」「2026」が締まる。
- ログイン: ライトエリア左上にブランド行（アプリアイコン＋「本荘キャンパス案内 AI」＋英字マイクロラベル）、
  白面には 4〜6% 不透明度のオーロラの気配（超低彩度・低速ドリフト、reduced-motion で静止）、
  シート内の「ログインはこちら」全幅ボタンはセグメントコントロール等の明快な UI へ。
  PC ではライトエリアの見出しを大きく（〜text-6xl）、余白を「意図した静けさ」に見せる。
- チャット: ヘッダーは無彩に静める。空状態はグラデ挨拶＋整ったサジェストカード（矢印装飾は削る）。
  入力バーは Gemini 風の浮遊ピル（下辺ボーダーで区切らず、余白とフェードマスクで浮かせる）。
  送信ボタンは有効時に白 or 単色（虹をやめる）。出典チップ・サイドバーは無彩の面と余白で整える。

**繰り返す: 以上は参考案。** Design Arena 首位の審美眼で、§0 の利用者の言葉と §2 の制約を守る限り、
これより良い解を自由に採ってよい。

## 5. モーション要求（「何を」= MUST、「どう」= Sol の裁量）

存在すべき演出（実装の仕方・味付けは自由）:

1. ログイン画面の初回入場シーケンス（要素が段階的に現れる。全体 1 秒以内に静定）
2. ログイン成功 → チャットへの画面遷移トランジション（ぶつ切り禁止。router の `<RouterView>` を
   `<Transition>` でラップする等。App.vue の変更は可）
3. チャット空状態の入場（挨拶・チップのスタッガー等）
4. 送信時のユーザー吹き出し出現アニメーション
5. モバイルドロワーの開閉（オーバーレイとパネルの滑らかな連動。現状 200ms 一律から質感を上げる）
6. hover / active / focus のマイクロインタラクション統一 — duration・easing をトークン化して全体に適用
7. 上記すべて `prefers-reduced-motion` で無効化（既存のグローバル抑止ルールを活かす）

推奨: 入力ピルの focus 遷移、スレッド切替時のメッセージ領域クロスフェード、ダイアログ入退場の磨き込み、
送信ボタンの有効化トランジション。

禁止: リング以外の常時ループ演出の多用（ログイン白面の超低彩度ドリフトのような「気配」レベルは可）、
300ms を超えて操作をブロックする遷移、視差スクロール等の重い効果、レイアウトシフトを起こす演出。

## 6. 実装ノート

- ブランチ: `feature/ui-restyle-sol`。コミットはローカルまで（push・PR 作成は不要。マージは Fable が行う）。
- 変更範囲: `frontend/` のみ。`docs/UI_POLISH_V2.md` への設計ノート追記（§7-4）は可。
- トークン集約: 色・radius・shadow・duration・easing は `tailwind.config.js` と `style.css` の CSS 変数に
  集約し、コンポーネント内のマジックナンバー散在を残さない。FR-9 で導入した ink/edge/fill/brand トークンは
  再設計してよい（名前含む）。
- `index.html` の `theme-color` 等メタ調整も対象。favicon はアプリアイコンのまま。
- 動作確認環境（ライブスタックと衝突しないこと）:
  - backend: `AGENT_MODE=mock MOCK_STATUS_DELAY_SECONDS=2.5 uvicorn app.main:app --port 8081`
    （リポジトリ直下で `python -m venv` 等は不要。既存の実行方法は backend/README または compose を参照）
  - frontend: `VITE_API_PROXY_TARGET=http://127.0.0.1:8081 npm run dev -- --port 5174`
  - ※ 5173/8080 は本番 compose が使用中。使わないこと。
- 検証ビューポート: 390×844 / 768×1024 / 1440×900 の 3 幅 × 主要状態（ログイン・空チャット・会話中・
  ストリーミング中・ドロワー/サイドバー・ダイアログ）。
- `npm test` / `npm run build` を完了報告前に必ず実行。

## 7. 検収（評価ループ — FR-9 §5 方式を継承・基準引き上げ）

1. 実装完了後、評価エージェント（実装と別人格）が mock 環境を Playwright で操作・撮影し、下のルーブリックで採点。
2. ルーブリック: FR-9 §4 の 10 観点 × 10 点。ただし観点 1 を「第一印象 — **『AI が作った量産 UI』感の脱却**を
   最重要指標とする」に改める。観点 6（マイクロインタラクション）は §5 の MUST 7 項の充足を前提とする。
3. **合格: 93 点以上**（FR-9 は 91 点合格だったが、その状態を利用者が「いかにも AI が作った UI」と評した。
   よって基準を引き上げる）。
4. 不合格なら減点を「観点・画面・具体箇所・修正提案」形式で Codex / Sol に差し戻し、最大 3 巡。
   合格後、Sol は §4 に対する「Sol 設計ノート」（採用した概念・色/タイポ/モーションのトークン表・
   推奨を破った箇所と意図）を本ドキュメント末尾に追記する。
5. 最終の採点表・スクリーンショット所見・設計ノートを Fable がレビューして merge。

## 8. Sol 設計ノート（実装後に Sol が追記）

### 8.1 採用概念 — Campus Signal

「大学の案内所」と「当日のウェイファインディング」をデザインの核にした。ログインは来場者を迎える生成りの
紙面、チャットは情報を読み込む暖色寄りの黒鉛面とし、ガラスカードと虹色発光に頼る汎用 AI チャットの語彙を
排した。ブランドカラーは単色の Signal Coral を現在地・操作開始・選択状態にだけ置き、Aurora は
LoadingSpinnerV5、空状態の利用者名、streaming 中の入力欄の 3 箇所に限定した。

### 8.2 デザイントークン

| 分類 | トークン | 値 / 用途 |
|---|---|---|
| Neutral | canvas / panel / raised / high | `#0D0F0E` / `#131614` / `#1B1E1C` / `#252925` |
| Light | paper / paper-ink | `#F0EFE9` / `#171916`（ログインのライトエリアと主要 CTA） |
| Text | primary / muted / dim | `#F2F1EC` / `#A9ADA7` / `#777C76`（canvas 比 17.01 / 8.45 / 4.51:1） |
| Accent | signal / signal-soft | `#FF7657` / `#FF9A80`（単色の案内標識） |
| Aurora copy | warm / bridge / mint | `#FF8F70` / `#FFC46B` / `#6FE8A8`（空状態の名前のみ） |
| Typography | display | 同梱 `Space Grotesk Variable`。英字・数字・見出し用。和文は Noto Sans JP 系スタック |
| Radius | sm / md / lg / sheet | `10 / 16 / 24 / 40px` |
| Shadow | hairline / raised / overlay | 内側 1px / 低く広い浮上影 / ドロワー・ダイアログ用の深い影 |
| Motion | fast / base / slow / stagger | `140 / 220 / 480 / 60ms` |
| Ambient | atmosphere | `16s`。ログインの単色低彩度ハローのみ。操作を妨げない |
| Easing | standard / expressive | `cubic-bezier(.2,.8,.2,1)` / `cubic-bezier(.16,1,.3,1)` |

Outfit は採用せず、ディレクトリごと削除した。全フォントは自己ホストで、実行時ネットワークアクセスはない。
`prefers-reduced-motion` では入場、遷移、stagger、hover の移動、ambient drift を停止し、タイプライターは既存ロジックで
フレーズ 1 の静的表示へ切り替わる。

### 8.3 画面への適用

- ログイン: 2 層構造を維持し、ライトエリアを編集的なブランド面、下部を Visitor Desk として再構成した。
  PC は説明とフォームを横並びにして余白へ役割を与え、モバイルは 390px の縦リズムに合わせて直列化した。
- チャット空状態: 中央の発光ロゴを主役にせず、左揃えの挨拶、日程、番号付き質問導線で「案内所」の世界観を作った。
- 会話中: AI 回答は中央ドキュメント、利用者発言は同カラム右端の graphite bubble。出典、サイドバー、ダイアログは
  無彩色の elevation と余白で整理した。
- 入力欄: 下辺の区切り線をなくした floating composer。通常 focus は無彩色、streaming 中だけ Aurora の細い縁を出す。
- モーション: ログイン初回入場、route 遷移、空状態 stagger、利用者吹き出し、ドロワー、統一 hover/focus/active の
  FR-12 §5 MUST 7 項を CSS / Vue Transition だけで実装した。

### 8.4 Fable 推奨から調整した箇所

- ログイン背景の「オーロラの気配」は多色 drift にせず、Signal Coral 1 色・最大 8% のハローにした。
  ログインで Aurora を消費せず、チャットの「AI が動く瞬間」との意味差を明確にするため。
- 空状態のグラデーションは挨拶全文ではなく利用者名だけに限定した。日本語見出しの可読性と色の一撃を両立するため。
- アプリアイコンは大きな発光オーブにせず、角丸の小さな識別標として扱った。既存ロゴの多色を面積で抑え、
  単色アクセント体系と共存させるため。
- Gemini 的な丸い浮遊入力は採用したが、focus 時の mint glow は使わず無彩色の hairline とした。
  Aurora は streaming 中だけ現れるという意味規則を守るため。

`LoadingSpinnerV5.vue` は SVG、テーマ、keyframes、timing、morph、shimmer、`type="transition"` を含め差分なし。
