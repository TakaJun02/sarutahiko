# campus-guide-agent システム仕様書

- 版: v0.1（2026-07-11, Fable 起草）
- ステータス: ドラフト（実装開始可）

## 1. 目的

秋田県立大学 本荘キャンパスのオープンキャンパス 2026 来場者（主に高校生・保護者）が、
学部・学科、施設、アクセス、当日のイベント等について自然言語で質問すると、
学内ナレッジ（RAG）と Web 検索を組み合わせて正確に回答する AI エージェントを提供する。

## 2. 想定ユーザーと利用シーン

- **主**: オープンキャンパス来場者（高校生・保護者）。スマートフォンからの利用が中心（参考動画も 738×1454 の縦画面）。
- 回答トーン: 丁寧・親しみやすく、高校生にも分かる語彙。既定言語は日本語。
- 誤情報が進路選択に影響しうるため、**根拠（出典）を持たない断定をしない**こと。

## 3. 機能要件

### FR-1 チャット応答（Agentic RAG）
- ユーザーの質問に対し、エージェントが以下を自律的に判断して回答する:
  1. 質問の解釈・必要な情報の特定
  2. 学内ナレッジ（ベクトル検索）からの取得
  3. ナレッジで不足する場合の Web Search（例: 最新のイベント日程）
  4. 取得情報の十分性評価（不足なら再検索）
  5. 回答生成（出典付き）
- ナレッジにも Web にも根拠がない場合は「分からない」と答え、問い合わせ先（大学公式サイト等）を案内する。

### FR-2 推論ステップの実況通知
- 推論の各ステップで「今何をしているか」を短い日本語テキストでフロントに随時配信する
  （例: 「質問を分析しています…」「学内ナレッジを検索しています…」「Web で最新情報を確認しています…」「回答をまとめています…」）。
- 配信は SSE で行う。イベントスキーマは `docs/ARCHITECTURE.md` に定義。
- フロントはこのテキストをローディング演出の横に表示する（`docs/UI_LOADING_ANIMATION.md`）。

### FR-3 回答のストリーミング表示
- 回答本文はトークンストリーミングでフロントに配信し、逐次描画する。
  （参考実装の「単語ごと 40ms 表示」は日本語と相性が悪いため、実ストリーミングに置き換える。）
- 回答は Markdown としてレンダリングする。

### FR-4 会話の継続
- スレッド（会話履歴）を保持し、文脈を踏まえた応答を行う。

### FR-5 ローディング演出
- Ver1.0: guidanceLLM2 の演出を完全再現（詳細仕様: `docs/UI_LOADING_ANIMATION.md`）。
- Ver5.0: Ver1.0 を土台に大幅アップデート（方針は同ドキュメント。詳細仕様は Fable が別途起草）。

### FR-6 ログイン（2026-07-11 追加）
- パスワードなしのニックネーム登録制ログイン（詳細仕様: `docs/UI_LOGIN.md`）。
- ログイン画面は ChatGPT / Gemini アプリを参考にしたスタイリッシュなダーク UI。
- セッショントークンで API を保護し、会話スレッドをユーザーに紐づける。

## 4. 非機能要件

- **NFR-1**: LLM 推論はローカル GPU（RTX 3090 Ti 24GB）上の vLLM のみを使用。外部 LLM API に依存しない。
- **NFR-2**: 最初のステータス通知は送信後 1 秒以内に返す（体感応答性）。
- **NFR-3**: スマートフォン縦画面を第一とするレスポンシブ UI。
- **NFR-4**: docker compose で全コンポーネントを起動できること。
- **NFR-5**: `prefers-reduced-motion` を尊重する（Ver5.0 演出で必須）。

## 5. スコープ外（現時点）

- ユーザー認証・個人情報の保存
- 多言語対応（構造上は拡張可能にしておくが、初期リリースは日本語のみ）
- 地図ナビゲーション機能（guidanceLLM2 の GUIDANCE MAP 相当は対象外。あくまでチャット）
